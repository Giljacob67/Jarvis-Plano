#7 - Jarvis Fase 7 — Browser Automation com Playwright + Approval Gates
What & Why
Adicionar navegação e automação real de navegador ao Jarvis usando Playwright, com foco em tarefas supervisionadas e seguras: abrir sites, navegar páginas, preencher formulários, capturar dados, baixar arquivos e executar fluxos guiados. Toda ação sensível ou irreversível deve passar por aprovação explícita. O objetivo é aproximar o Jarvis de um agente estilo OpenClaw sem perder controle, auditabilidade e segurança. As Fases 1–6 devem continuar intactas.

Done looks like
O Jarvis consegue abrir páginas em um navegador controlado e executar steps básicos de automação
/playbooks passa a incluir workflows com browser automation
/approve e /reject controlam ações sensíveis vindas de fluxos web
O agente consegue:
- abrir URL
- fazer screenshot
- extrair texto visível
- clicar em elementos
- preencher campos
- selecionar opções
- baixar arquivos
- esperar navegação de forma robusta
Toda ação web fica registrada em ActionLog
Toda sessão de navegador é rastreada no banco
O agente pede aprovação antes de:
- enviar formulário
- clicar em ações potencialmente irreversíveis
- confirmar compras, pagamentos, exclusões, submissões finais
O agente nunca tenta contornar CAPTCHA, 2FA, paywall ou mecanismos de segurança
Quando login manual for necessário, o fluxo pausa e orienta o usuário
Existem comandos Telegram para iniciar, acompanhar, aprovar e encerrar sessões de navegador
Todos os testes novos passam; 149 testes anteriores continuam passando

Out of scope
Bypass de CAPTCHA, 2FA ou qualquer controle de segurança
Compras automáticas, pagamentos ou transferências sem aprovação
Execução destrutiva silenciosa
Navegação invisível para burlar sites
Múltiplos usuários simultâneos
Browser em dispositivo do usuário
Extensão de navegador
RPA desktop fora do browser
WhatsApp/Slack/Discord

Constraints
Usar Playwright Python oficial
Executar browser automation apenas em allowlist de domínios configuráveis
Toda sessão deve ser efêmera e isolada
Não armazenar senhas em texto puro
Não salvar cookies persistentes por padrão
Permitir pausa para login manual quando necessário
Toda ação sensível exige PendingApproval
Toda navegação deve ter timeout configurável
Toda resposta ao usuário em pt-BR
Respeitar robots, segurança e controles dos sites; não implementar evasão
User_id sempre String
Timezone padrão America/Sao_Paulo
Todos os testes sem chamadas externas reais; mockar Playwright
