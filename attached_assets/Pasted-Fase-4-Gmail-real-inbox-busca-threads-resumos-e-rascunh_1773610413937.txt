Evolua o projeto existente sem recriar o scaffold e sem quebrar as fases anteriores.

Objetivo
Adicionar integração real com Gmail para 1 usuário com foco em produtividade:
- listar e-mails importantes
- buscar e-mails com filtros
- abrir threads para contexto
- resumir inbox
- criar rascunhos
- criar rascunhos de resposta
- enviar e-mail somente por ação explícita

Nesta fase, NÃO implementar:
- apagar e-mails
- mover para lixo
- marcar como lido/não lido
- arquivar
- labels customizadas
- push notifications do Gmail

Escopo funcional desta fase
1. Gmail OAuth integrado ao fluxo Google já existente
2. leitura real de mensagens e threads
3. comandos Telegram para inbox e rascunhos
4. tools da OpenAI para resumir, buscar e rascunhar
5. envio real apenas por comando explícito e com confirmação simples
6. tudo em pt-BR

6) Comandos Telegram
Adicionar estes comandos:

- /inbox
  lista 5–10 e-mails recentes/importantes

- /emailsearch <consulta>
  usa query do Gmail API
  exemplo:
  /emailsearch from:cliente@empresa.com.br is:unread

- /thread <thread_id>
  mostra resumo da thread

- /drafts
  lista rascunhos recentes

- /draftemail <para> | <assunto> | <corpo>
  cria rascunho novo

- /replydraft <message_id> | <corpo>
  cria rascunho de resposta dentro da thread da mensagem

- /senddraft <draft_id>
  envia um rascunho explicitamente

- /inboxsummary
  faz resumo da inbox recente

Regras:
- parsing robusto com mensagens de erro claras em pt-BR
- /senddraft só envia por comando explícito
- antes de enviar, responder com um resumo curto do draft quando útil
- se Gmail não estiver conectado, orientar para /connectgoogle
