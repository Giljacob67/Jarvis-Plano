4) Rotas OAuth
Implementar rotas reais:
- GET /auth/google/start
- GET /auth/google/callback
- GET /auth/google/status
- POST /auth/google/disconnect

Comportamento:
- /auth/google/start gera URL OAuth e redireciona o usuário
- usar state para segurança
- /auth/google/callback troca authorization code por tokens
- persistir refresh token
- associar a conta Google ao usuário único do projeto
- /auth/google/status informa se Google está conectado, escopos e validade básica
- /auth/google/disconnect remove tokens locais
